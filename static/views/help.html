Welcome to DEO

DEO is a powerful automation platform that helps you build Slack bots, manage teams, and track metrics through customizable dashboards.

Workspaces

Connect your Slack workspace to start building automations.

Bot Templates

Create automated workflows that respond to triggers and commands.

Key Features

Quick Start Guide

Get up and running with DEO in just a few steps.

Step 1: Connect Your Workspace

Before you can start building bots, you need to connect a Slack workspace.

  1. Navigate to Workspaces in the sidebar
  2. Click + Create to add a new workspace
  3. Enter your Slack Bot Token (starts with xoxb-)
  4. Give your workspace a name and save
You can find your Bot Token in your Slack App settings under OAuth & Permissions.

Step 2: Create Your First Bot

Now let's create a simple bot that sends a message to a Slack channel.

  1. Click Canvas in the sidebar to open the bot builder
  2. Select your workspace from the dropdown
  3. Drag a Message block onto the canvas
  4. Connect it to the Trigger block
  5. Configure the message: select a channel and enter your message
  6. Click Save & Run to execute

Step 3: Create a Team

Teams help you organize Slack members for dashboards and targeted messaging.

  1. Go to Teams in the sidebar
  2. Click + Create
  3. Enter a team name and select a workspace
  4. Add members from your Slack workspace

Step 4: Build a Dashboard

Track your team's performance with custom metrics.

  1. Navigate to Dashboards
  2. Click + Create
  3. Link it to a team and define your metrics (e.g., sales, calls)
  4. Share the dashboard URL with your team

Slack App Setup

Create a Slack app using an app manifest for quick and easy setup. The manifest pre-configures all required permissions and features.

What is an App Manifest?

An app manifest is a JSON configuration file that defines your Slack app's settings, permissions, and features. Using a manifest allows you to:

  • Skip manual configuration of OAuth scopes
  • Ensure all required permissions are set correctly
  • Quickly replicate the app across workspaces
  • Version control your app configuration

Creating Your Slack App

  1. Go to api.slack.com/apps and click Create New App
  2. Select From an app manifest
  3. Choose your workspace and click Next
  4. Select JSON as the format
  5. Paste the manifest below and click Next
  6. Review the configuration and click Create

DEO App Manifest

Copy this manifest to create your DEO Slack app:

{
  "display_information": {
    "name": "DEO",
    "description": "Onboarding and metrics tracking bot for marketing teams",
    "background_color": "#2c3e50"
  },
  "features": {
    "app_home": {
      "home_tab_enabled": false,
      "messages_tab_enabled": true,
      "messages_tab_read_only_enabled": false
    },
    "bot_user": {
      "display_name": "DEO",
      "always_online": true
    }
  },
  "oauth_config": {
    "scopes": {
      "bot": [
        "app_mentions:read",
        "assistant:write",
        "channels:history",
        "channels:read",
        "chat:write",
        "chat:write.customize",
        "chat:write.public",
        "files:read",
        "files:write",
        "groups:history",
        "groups:read",
        "im:history",
        "im:read",
        "im:write",
        "im:write.topic",
        "mpim:history",
        "pins:write",
        "remote_files:read",
        "remote_files:share",
        "remote_files:write",
        "search:read.files",
        "users.profile:read",
        "users:read",
        "users:read.email"
      ]
    }
  }
}

After Creating the App

  1. In your new app's settings, go to OAuth & Permissions
  2. Click Install to Workspace
  3. Authorize the app when prompted
  4. Copy the Bot User OAuth Token (starts with xoxb-)
  5. Use this token when creating a workspace in DEO
Keep your Bot Token secure. Never share it publicly or commit it to version control.

Included Permissions

The manifest includes all permissions required for DEO features:

  • channels:read, channels:history - Read channel info and messages
  • chat:write, chat:write.public - Send messages to channels
  • files:read, files:write - Upload and access files
  • im:read, im:write, im:history - Direct message capabilities
  • users:read, users.profile:read - Access user information
  • groups:read, groups:history - Private channel access

Understanding Workspaces

Workspaces are the foundation of DEO. They connect your Slack workspace to enable bot automation and team management.

What is a Workspace?

A workspace in DEO represents a connection to a Slack workspace. Each workspace stores:

  • Your Slack Bot Token for API access
  • Workspace metadata (name, team info)
  • Associated bot templates, teams, and dashboards

Why Multiple Workspaces?

You might need multiple workspaces if:

  • You manage multiple Slack organizations
  • You have separate workspaces for different departments
  • You want to keep production and testing environments separate

Required Slack Permissions

Your Slack bot needs these OAuth scopes:

channels:read - View channels
channels:history - Read channel messages
chat:write - Send messages
files:write - Upload files
users:read - View user info

Setting Up a Workspace

Follow these steps to connect your Slack workspace to DEO.

Creating a Slack App

  1. Go to api.slack.com/apps
  2. Click Create New App
  3. Choose From scratch
  4. Name your app and select your workspace

Configuring OAuth Scopes

  1. In your Slack app, go to OAuth & Permissions
  2. Under Scopes, add the following Bot Token Scopes:
channels:read, channels:history, chat:write,
files:write, files:read, users:read,
groups:read, groups:history, im:write,
conversations.open

Installing to Workspace

  1. Click Install to Workspace
  2. Authorize the app
  3. Copy the Bot User OAuth Token (starts with xoxb-)
  4. Paste it into DEO when creating your workspace
Keep your Bot Token secure. Never share it publicly.

What are Bots?

Bots are automated workflows that perform actions in your Slack workspace based on triggers and conditions.

Bot Components

Every bot consists of:

  • Trigger - What starts the bot (manual, scheduled, or scan)
  • Blocks - Actions the bot performs (message, scan, await, response)
  • Connections - The flow between blocks

Common Use Cases

Daily Standup Reminders

Schedule a bot to message your team every morning asking for updates.

Command-Based Actions

Scan for commands like --report and respond with relevant data.

Metrics Collection

Ask team members for their daily metrics and await their responses.

File Distribution

Send files to channels or users automatically.

Creating a Bot

Use the visual Canvas to build your bot workflow by connecting blocks.

Using the Canvas

  1. Click Canvas in the sidebar
  2. Select a workspace from the dropdown (required)
  3. Give your bot a name in the input field at the top
  4. Drag blocks from the left panel onto the canvas
  5. Click and drag from connection points to link blocks
  6. Click a block to configure its settings

Configuring the Trigger

Every bot starts with a Trigger block. Configure it as:

  • Manual - Run the bot manually via Save & Run
  • Scheduled - Run automatically using cron syntax
Example cron: 0 9 * * 1-5 runs at 9 AM on weekdays.

Saving and Running

  • Save - Saves the bot template without running it
  • Save & Run - Saves and immediately executes the bot
  • Reset - Clears the canvas to start fresh

Understanding Blocks

Blocks are the building blocks of your bot workflows. Each block performs a specific action.

Message Block

Sends a message to a Slack channel or directly to users.

  • Channel Mode - Send to a specific channel
  • Users Mode - Send DMs to selected users
  • File Attachment - Optionally attach a file

Scan Block

Monitors a channel for a specific command and triggers subsequent blocks when found.

  • Channel - The channel to monitor
  • Command - Text to look for (e.g., --deploy)
  • Interval - How often to check (e.g., 30s, 5m)
Scan blocks run continuously until manually stopped from the Bot Templates page.

Await Block

Pauses execution and waits for a response from users.

  • Expected Response - Optional text to match
  • Timeout - How long to wait (e.g., 24h)
  • Failure Message - Message if timeout is reached

Response Block

Sends a follow-up message after an Await block completes.

  • Uses the same channel/user context as the preceding Message block
  • Typically used to confirm receipt or provide next steps

Triggers & Scheduling

Control when your bots run using different trigger types.

Manual Triggers

Manual triggers require you to click Save & Run or use the Run button on the Bot Templates page. Useful for:

  • Testing new bots
  • One-time announcements
  • On-demand actions

Scheduled Triggers

Scheduled triggers use cron syntax to run automatically. Format:

┌───────────── minute (0-59)
│ ┌─────────── hour (0-23)
│ │ ┌───────── day of month (1-31)
│ │ │ ┌─────── month (1-12)
│ │ │ │ ┌───── day of week (0-6, Sun-Sat)
│ │ │ │ │
* * * * *

Common Cron Examples

0 9 * * * Every day at 9:00 AM
0 9 * * 1-5 Weekdays at 9:00 AM
0 */2 * * * Every 2 hours
0 9 1 * * First day of every month at 9 AM
30 17 * * 5 Every Friday at 5:30 PM

Managing Teams

Teams help you organize Slack members for dashboards, targeted messaging, and access control.

What are Teams?

Teams are groups of Slack users that you create within DEO. They are used for:

  • Linking to dashboards for metrics tracking
  • Sending targeted messages to specific groups
  • Managing access to dashboard submissions

Creating a Team

  1. Navigate to Teams in the sidebar
  2. Click + Create
  3. Enter a team name
  4. Select the workspace this team belongs to
  5. Click Create

Team Details

Once created, you can view and manage:

  • Team Info - Name, ID, workspace, member count
  • Members - List of team members with their Slack info
  • Actions - Add members, create hiring links, delete team

Adding Team Members

Add Slack users to your teams using multiple methods.

Adding from Slack

The easiest way to add members is directly from your Slack workspace:

  1. Open a team and click Add Member
  2. Switch to the From Slack tab
  3. Select users from the list (shows all workspace members)
  4. Click Add Selected

Manual Entry

You can also add members manually by providing their details:

  1. Click Add Member
  2. Stay on the Manual tab
  3. Enter the member's name and email
  4. Optionally enter their Slack User ID
  5. Click Add Member
You can find a user's Slack ID by clicking their profile in Slack and selecting "Copy member ID".

Removing Members

To remove a member from a team:

  1. Open the team details
  2. Find the member in the Members card
  3. Click the X button next to their name

Creating Dashboards

Dashboards let you track team performance with custom metrics and visualizations.

What are Dashboards?

Dashboards are linked to teams and allow members to submit metrics on a regular basis. They provide:

  • Custom metric tracking (sales, calls, meetings, etc.)
  • Automatic aggregation and visualization
  • Leaderboards for team competition
  • Historical graphs and analytics

Creating a Dashboard

  1. Go to Dashboards in the sidebar
  2. Click + Create
  3. Enter a dashboard name
  4. Select or enter the Team ID (use "Pick Team" to choose)
  5. Define your metrics (comma-separated, e.g., sales, calls, meetings)
  6. Select a reporting period (daily, weekly, monthly)
  7. Click Create

Dashboard Views

Each dashboard has three main views:

  • Metrics View - Overview cards showing totals and the leaderboard
  • Graph View - Historical data visualization over time
  • Details View - Configuration, member access, and danger zone

Tracking Metrics

Learn how to define, collect, and visualize team metrics.

Defining Metrics

When creating a dashboard, define metrics as a comma-separated list:

sales, calls, meetings, emails sent

Each metric becomes a column that team members can submit values for.

Submitting Metrics

Team members submit metrics through the public dashboard URL:

  1. Share the dashboard URL with team members
  2. Members enter their passcode to access
  3. They fill in their metric values for the current period
  4. Click Submit to record their data
Passcodes are auto-generated and can be found in the dashboard's Details view under "Member Access".

Reporting Periods

  • Daily - Members submit every day, data resets daily
  • Weekly - Members submit once per week (Monday-Sunday)
  • Monthly - Members submit once per month

Leaderboards

Leaderboards rank team members based on their metric submissions.

How Rankings Work

The leaderboard ranks members by their total submission value. Rankings consider:

  • Sum of all metric values for the current period
  • Members with no submissions appear at the bottom
  • Ties are broken by submission time (earlier is higher)

Viewing the Leaderboard

The leaderboard is displayed on the Metrics View of each dashboard. It shows:

  • Rank position
  • Member name
  • Individual metric values
  • Total score

Filtering by Member

Use the member filter dropdown to view metrics for a specific team member instead of the aggregate view.

Scan Blocks

Scan blocks monitor Slack channels for specific commands and trigger actions when detected.

How Scanning Works

  1. The scan block periodically checks the channel history
  2. When the command is found in a new message, subsequent blocks execute
  3. The scan continues running, waiting for the next occurrence
  4. Scanning runs until manually stopped

Configuration Options

  • Channel - Which channel to monitor
  • Command - Text pattern to match (e.g., --status)
  • Interval - Check frequency (30s, 1m, 5m, 30m, 1h)

Example: Command Bot

Create a bot that responds to --report with a file:

  1. Add a Scan block connected to the Trigger
  2. Set the channel and command (--report)
  3. Add a Message block connected to the Scan block
  4. Configure the Message to send to the same channel with a file attached
  5. Save & Run to start scanning

Stopping Scans

To stop a running scan:

  1. Go to Bot Templates
  2. Find your template (it will show "Scanning" status)
  3. Click Stop to halt the scan

Await & Response Blocks

Create interactive bots that wait for user responses before continuing.

The Await Block

The Await block pauses bot execution and waits for replies:

  • Monitors the channel/DM where the previous message was sent
  • Waits for responses from all relevant users (channel mode) or first response (DM mode)
  • Can filter for specific response text
  • Has a configurable timeout with failure message

Configuration Options

  • Expected Response - Optional text the response must contain
  • Timeout - How long to wait (e.g., 1h, 24h, 7d)
  • Failure Message - Message sent if timeout is reached

The Response Block

The Response block sends a follow-up message after the Await completes:

  • Automatically uses the same channel context
  • Typically confirms receipt or provides next steps
  • Only executes if the Await was successful (not timed out)

Example: Feedback Collection

  1. Message block asks users "Please reply with your feedback"
  2. Await block waits 24 hours for responses
  3. Response block sends "Thanks for your feedback!"

Job Applications

Create hiring workflows with application forms and manage candidates.

Creating Application Links

Generate unique application URLs for job positions:

  1. Open a team and click Hire
  2. Enter the position title
  3. Copy the generated application URL
  4. Share the URL with candidates

Application Form

Candidates who visit the application URL will see a form to submit:

  • Full name
  • Email address
  • Position applying for
  • Additional information

Managing Applications

View all application links by clicking View Links on the team page. From there you can:

  • Copy application URLs
  • See how many applications each link has received
  • Deactivate or delete links
